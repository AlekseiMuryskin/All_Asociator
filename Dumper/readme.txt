Dump.py - скрипт, который достает из БД записи событий, которые нашел RecStream.py.
Для работы достаточно:
1) Заполнить конфиг: параметры БД с записями о событиях, параметры сервера ArcLink, параметры арзива: объект (такой же как в настройках RecStream), время начала/конца для выбора событий, путь до папки с архивом, Archive_dt - время в с, до события и после (если 5 с, то файл будет 10), список станций в виде файла.
2) Заполнить список станций для выгрузки (stat.txt): построчно, образец BR.B209.*.EHZ
3) запустить скрипт, сейскомп не нужен. python DumpSgrm.py
ВНИМАНИЕ! в конфиге время начало/конца обращается не к времени события, а к времени последнего ОБНОВЛЕНИЯ информации о нем. То есть если данные отставали на неделю и вчера пришли, появилось новое событие недельной давности (или обновилось старое), то при запросе к вчерашнему дню скрипт его достанет, потому что информация в БД была обновлена вчера.
